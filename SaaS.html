SaaS: One Size Fits All…


What?

Yeah…you read it right. Being a Software as a Service product it needs to serve a wide variety of functionality for multiple teams with a handful of UI components. In lack of better words it’s more like Legos. The UI components are simple but when used in combination it can tailor to a wide range of user needs.

Why?

No one asked for it to change. But while I was supporting the current application I felt a need to give it a revamp. Some of the workflows were cumbersome to the users, some of the actions weren't obvious and moreover the accomplishments of the users were not celebrated.

How?

So in 3 weeks, I went pixel by pixel to see what can be changed and spent the time in getting to know what users want? What are their pain points? Which user flow makes more sense? Tried to identify the redundant tasks to automate things.(Make the computer work for you then you work for the computer).

Results

A complete revamp of the application from front end and back end and every possible end.
